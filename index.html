Overview

A benchmark for recovering small everyday objects from indoor videos.

PinpointQA evaluates whether models can detect a target, ground it against nearby references, describe its location, and organize the result into a usable spatial output.

Focus

Target-driven recovery

Designed around practical object retrieval rather than generic scene understanding.

Input

Indoor videos

Captures clutter, occlusion, long temporal gaps, and weak visual saliency.

Output

Actionable spatial cues

Measures whether model outputs can directly support downstream search.

Tasks

4-stage capability chain

From appearance verification to structured spatial prediction.

PinpointQA Task Formulation

Four compact tasks define the full recovery pipeline.

TPV

Check whether the target object appears in the video.

NRI

Identify the nearest reference object around the target.

FSD

Describe the target location with concise spatial language.

SSP

Return the location as structured, directly usable spatial fields.

Data Construction Pipeline

From indoor scenes to grounded, target-centric QA pairs.

PinpointQA starts from richly annotated indoor datasets, constructs target-centered spatial relations, and turns them into aligned question-answer samples across the four tasks.

Figure 2. The pipeline filters candidate targets, computes local spatial relations, and generates task-aligned QA annotations.
